## What is Express Framework?

**Express.js** is a **minimal and flexible web application framework for Node.js**.  
It makes server-side development easier by providing simple methods for handling requests, responses, routing, and middleware.

### Why use Express?

- Reduces complex server code
- Makes APIs and web servers easy to create
- Widely used in real-world backend applications

## Creating a Basic Express Server

Example of a simple Express server:

```
const express = require("express");
const app = express();
const port = 3000;

app.use("/test", (req, res) => {
    res.send("Server started");
});

app.use("/main", (req, res) => {
    res.send("Another route");
});

app.listen(port, () => {
    console.log("Server started running on port " + port);
});
```

Explanation

- `express()` creates an application instance

- `app.use()` defines routes

- `app.listen()` starts the server

- Server runs at: `http://localhost:3000`


## Understanding the -g Flag in npm
### What does -g mean?

- -g stands for global installation

### Local Installation (default)
```
npm install <package>
```

- Installed inside the project
- Used only in that project
- Added to package.json

### Global Installation
```
npm install -g <package>
```
- Installed system-wide
- Can be used from any directory
- Mostly used for CLI tools

Example
```
npm install -g nodemon
```
